Assignment Overview: Process Modeling and Visualization

Estimated time: 4 minutes
Introduction

In this module, you will develop essential process modeling and visualization skills to support Retail Bank's Digital Core Transformation. You'll learn to map customer onboarding and compliance workflows, analyze system pain points, and produce clear visual documentation using standard UML techniques for enterprise banking environments.

This lab further strengthens your modeling skills by guiding you through the customer onboarding and KYC verification process while ensuring adherence to UML standards and accuracy when representing workflow steps and system interactions.
Tasks
Task 1: Model customer onboarding and KYC verification

Design a flowchart capturing Retail Bank's end-to-end customer onboarding journey, from account request through KYC validation and account creation. Use standard UML symbols to depict steps, decisions, and dependencies, and highlight operational bottlenecks and opportunities for improvement.
Task 2: Define key process components

Break down the onboarding workflow into critical phases, outlining significant actions and decision points. Document each stage for clarity, emphasizing system checks, manual compliance reviews, and automated API interactions that impact account setup and configuration.
Task 3: Build the onboarding flowchart using UML conventions

Create the visual flowchart in Lucidchart or a similar tool, ensuring accuracy in representing the workflow and consistent use of UML conventions. Include annotations for decision logic or data movements where relevant.
Task 4: Develop use case diagrams for system interactions

Construct a use case diagram capturing stakeholder roles—retail customers, branch staff, compliance officers, and IT admins—and their interactions with Retail Bank's core banking system. Link each scenario to functional and non-functional requirements documented earlier for comprehensive system analysis.
Task 5: Review and refine the flowchart for accuracy

Validate the flowchart by checking for completeness, efficiency, and clarity. Remove redundant steps, confirm decision paths, and ensure the final model is understandable for all stakeholders.

Introduction

Flowcharts are vital tools for solutions and systems architects, enabling the visualization of complex processes, such as customer onboarding and KYC verification. This lab guides you through creating a flowchart to map the process from account request to account creation confirmation, highlighting decision points and data trends, such as KYC document validation. A well-designed flowchart enhances stakeholder understanding, identifies inefficiencies, and supports process optimization, aligning with PMI standards for effective project delivery.
Scenario overview

A mid-sized retail bank operates on legacy banking platforms, resulting in inefficiencies during onboarding and poor customer satisfaction. The project aims to implement a unified core banking system, utilizing a MySQL database and Tableau dashboards, streamline onboarding, ensure real-time updates, and comply with RBI regulations for the current quarter. This lab focuses on modeling the customer onboarding and verification process to improve operational efficiency.

Task 3: Create the flowchart

Design a flowchart to visualize the onboarding and KYC verification process.
Key steps

Use Lucidchart or any other diagramming tool, for example, Microsoft Visio, Draw.io, or Creately, to create a flowchart that illustrates the appointment scheduling process for a retail bank scenario.
Step 1: Create outline

To create a flowchart outline, begin with stages such as Request Account, Validate KYC and Credit Score, Create Account, and Notify Customer.
Step 2: Instructions for creating Lucidchart

    Log in to Lucidchart and open a new blank document

    From the left side bar, select Shapes to open the Shapes Panel, and select Flowchart for UML shapes (ovals, rectangles, diamonds, arrows). Drag
        Ovals for Start/End
        Rectangles for Actions, such as Customer Logs In, Select Account Type, Branch, Create Account, and Request Additional Documents
        Diamond for decision points, such as Is KYC complete?

Step 3: Customize flowchart using colors

Adding colors and line styles distinguishes the flowchart between actions and decision points.

    Green color for action steps
    Yellow color for decision points
    Blue color for start and end

Step 4: Validate the flowchart

Ensure the flowchart accurately represents the process and is efficient. Validate the created flowchart using the checklist below:

    Ensure that flowcharts adhere to UML standards with clear representation
    All decision points have valid Yes/No paths
    Accurately reflect customer data trends, for example, verification success rates
    Combine redundant or repetitive actions wherever possible
    Review each step for completeness, for example, authentication, KYC status updates

Step 5: Save the flowchart

Once completed, export the flowchart as a PNG file named Flowchart.png and keep it handy for the final project.
